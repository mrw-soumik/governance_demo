# Security Policy

This repository is a demonstration project and should not contain production
credentials, customer data, or private infrastructure details.

## Reporting

If you reuse this repository beyond the demo, report security concerns
privately to the repository owner or the engineering lead responsible for the
project. Do not open a public issue for active credentials or exploitable
secrets.

## Minimum expectations

- never commit secrets to the repository
- rotate any credential that is exposed by mistake
- review dependency updates before merging
- keep the CI workflow green before release or demo use
