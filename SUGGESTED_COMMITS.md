# Suggested Commit Sequence

Use these commits so the history looks clean in your presentation.

1. `Initial commit: add startup governance demo API`
2. `Add tests for health and projects endpoints`
3. `Add project status filtering`
4. `Add GitHub Actions CI pipeline`
5. `Improve README with problem statement and governance controls`

## Suggested Demo Flow

1. Show repository home page
2. Show commit history
3. Show Actions / CI runs
4. Optionally show `/docs` locally
